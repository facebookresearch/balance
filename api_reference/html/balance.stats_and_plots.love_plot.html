balance.stats_and_plots.love_plot¶

Love plot — visual ASMD diagnostic.

A “Love plot” (after Thomas Love) is the canonical visual for showing how much each covariate’s ASMD shrinks after applying weights. balance did not previously expose one; this is the first such helper. Reference: R’s cobalt::love.plot.

The function is split into a primitive (love_plot) operating on raw pd.Series inputs and a method shortcut (BalanceDFCovars.love_plot) that pulls before/after ASMD off a fitted BalanceFrame’s lineage.

The primitive accepts a single asmd_before series for the pre-adjust diagnostic case (when no “after” yet exists) and falls back to a single- series scatter then. With both asmd_before and asmd_after it draws the canonical before-vs-after view.

balance.stats_and_plots.love_plot.love_plot(asmd_before: Series, asmd_after: Series | None = None, *, threshold: float = 0.1, ax: Axes | None = None) → Axes[source]¶

Side-by-side scatter of per-covariate ASMD before vs. after weighting.

A “Love plot” (after Thomas Love) is the canonical visual for showing how much each covariate’s ASMD shrinks after applying weights. Reference: R’s cobalt::love.plot.

Parameters:

  • asmd_before – ASMD per covariate. In two-series mode (when asmd_after is supplied) this is the BEFORE-adjustment ASMD — e.g. the output of the unadjusted view’s BalanceDFCovars.asmd(). In single-series mode (when asmd_after is None) this is the only ASMD plotted — typically the current / weighted ASMD when no “before” yet exists (the pre-adjust diagnostic case). The argument name reflects the canonical two-series usage; the single-series branch labels the points as “ASMD” rather than “Unweighted” so a caller passing the weighted ASMD here is not mis-described in the legend.

  • asmd_after – ASMD per covariate with weights. When None the plot shows asmd_before only as a single-series scatter – this is the pre-adjust diagnostic case (no “after” yet exists), conceptually closer to asmd() than to asmd_improvement().

  • threshold – Vertical reference line, default 0.1 — the conventional “balance achieved” cutoff. Must be non-negative (ASMD itself is non-negative by construction).

  • ax – Optional matplotlib Axes to draw into. If None, a new figure is created sized to the number of covariates.

Returns:

The Axes used, returned for further customization (titles, legend, save, etc.).

Return type:

matplotlib.axes.Axes

Raises:

ValueError – If threshold is negative, or if asmd_before and asmd_after (when both supplied) share no covariates.

Examples

>>> import pandas as pd
>>> from balance.stats_and_plots.love_plot import love_plot
>>> before = pd.Series({"age": 0.42, "income": 0.31})
>>> after = pd.Series({"age": 0.05, "income": 0.08})
>>> ax = love_plot(before, after)
